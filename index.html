    Harvest

    Very early days!

    What?

    Harvest is a game that lets players see what it's like to run a business in the shared, limited resource of a fishing ground. Will players co-operate and fish sustainably, or go for short term profit and harvest the fish to extinction?

    Why?

    Three reasons: fun, learningâ€¦ and profit.

    Fun: Harvest is simple but entertaining to play.

    Learning: I wanted to build an app with a small problem domain to experiment with CQRS and Event Sourcing in Ruby. I also wanted to try building a rich-client browser app (I'll use Ember.js) against a simple API server (using Webmachine). This all makes it a big detour from The Rails Way, which means more work now, but will lead potentially to more powerful tools in future.

    Profit: The engine for Harvest could be used to build many other games and simulations. It could conceivably even be used to build management dashboard simulations. But these ideas are a long way off. Profit is not an immediate goal: this is an experiment first, a startup second.
